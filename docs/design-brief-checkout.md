# Промпт для Gemini — страница Checkout (оформление заказа)

> Скопировать всё ниже одним сообщением в Gemini. Продолжение проекта АланПир (осетинские пироги, Москва, доставка).

---

Спроектируй и свёрстай **полноценную страницу оформления заказа** для существующего проекта Next.js 15 + React 19 + Tailwind 4 (AI Studio applet «АланПир»). Используй уже заданный дизайн-язык проекта:

## Дизайн-контекст (не менять)

- Палитра: `--color-terracotta: #C04D29`, `--color-bg-warm: #FAF9F6`, `--color-ink: #1D1D1B`, `--color-muted: #6B6B6B`, `--color-surface: #FFFFFF`, `--color-border-warm: #E5E1DA`
- Шрифты: `Inter` (UI), `Playfair Display` (заголовки, через `.font-serif` / `font-family: var(--font-serif)`)
- Скругления: 16-24px на крупных блоках, 999px на чипах/пилюлях
- Карточки: `bg-surface rounded-2xl border border-border-warm`, без теней
- Акцентные кнопки: `bg-terracotta text-white rounded-xl` или `rounded-full`
- Иконки: `lucide-react`
- Лейаут: `max-w-[940px]` центрирован (как на других страницах проекта)
- Заголовок страницы — крупный (`text-4xl md:text-5xl`), обычно с italic-акцентом в `text-terracotta`

## Что должно быть на странице

Роут: `app/checkout/page.tsx`. Одна страница с двумя колонками на десктопе (`md:grid-cols-[1.3fr_1fr]`) и одной колонкой на мобилке. Слева — формы с шагами, справа — sticky-сайдбар с итогом корзины. Без переходов между роутами — всё на одной странице.

### Левая колонка — 4 секции одна под другой

Каждая секция — карточка `bg-surface rounded-2xl border border-border-warm p-6 md:p-8` с заголовком `text-xl font-serif`. Номер шага в круглом бейдже слева от заголовка.

#### Шаг 1. Контакты
- Поле «Имя» (`input`, обязательно)
- Поле «Телефон» (маска `+7 (___) ___-__-__`, обязательно)
- Поле «Email» (опционально, для чека)
- Если пользователь авторизован — поля предзаполнены, сверху плашка «Вы вошли как {имя}» со ссылкой «Выйти»
- Если не авторизован — под полями ссылка-кнопка «Войти, чтобы быстрее оформить» (открывает модалку авторизации)

#### Шаг 2. Способ получения
- Переключатель-сегмент из двух кнопок на всю ширину: **Доставка** / **Самовывоз** (активная — тёмная `bg-ink text-white`, неактивная — `bg-bg-warm border`)
- Если выбрана **Доставка**:
  - Блок «Адрес»:
    - Если есть сохранённые адреса — список радио-карточек (адрес, подъезд/этаж, кнопка «Редактировать»)
    - Кнопка-плейсхолдер «+ Добавить новый адрес» открывает модалку DaData (показать компактный превью внутри этой страницы: инпут с подсказками и разворачивающиеся поля квартира/подъезд/этаж/домофон/комментарий)
  - Блок «Когда доставить»:
    - Две радио-кнопки: **Как можно скорее** (~20–120 минут) и **Ко времени**
    - Если «Ко времени» — два `<select>` рядом: дата (сегодня/завтра/послезавтра) и слот времени (08:30, 09:00, ..., 19:00 шагом 30 минут)
  - Инфо-плашка мелко: «Доставка по Москве от 20 ₽/км. Минимальный заказ 2000 ₽. Бесплатно при заказе от 5000 ₽»
- Если выбран **Самовывоз**:
  - Карточка с адресом пункта выдачи: «Москва, ул. Тихвинская, 20», часы работы, мини-карта-плейсхолдер (квадрат 100%×200px с `bg-bg-warm rounded-xl`)
  - Плашка-напоминание: «При самовывозе скидка 10% (не суммируется с промокодами и сетами)»
  - Выбор времени самовывоза — те же две радио-кнопки

#### Шаг 3. Оплата
- Три крупные радио-карточки в строку (на мобилке в столбик):
  - **Банковская карта** — иконка карты, подпись «МИР, Visa, MasterCard»
  - **СБП** — иконка СБП, подпись «Оплата через банковское приложение»
  - **Наличные/карта курьеру** — иконка купюры, подпись «Оплата при получении»
- При выборе карточка обводится `border-terracotta border-2`
- Под выбором — мелкая юридическая плашка «Нажимая "Оформить заказ", вы соглашаетесь с офертой»

#### Шаг 4. Комментарий к заказу
- `textarea` на 3 строки, плейсхолдер «Дополнения, аллергии, пожелания курьеру...»
- Чекбокс «Не звонить, привезти молча» (полезно жестом)
- Чекбокс «Нужны приборы и салфетки» (по умолчанию выключен — экологично)

### Правая колонка — Sticky-сайдбар «Ваш заказ»

Карточка `sticky top-24 bg-surface rounded-2xl border border-border-warm p-6`. На мобилке становится обычным блоком снизу.

Содержимое сверху вниз:
1. Заголовок «Ваш заказ» + счётчик «3 позиции»
2. Список 2–3 товаров-заглушек (мини-строки: фото 48×48 с `rounded-lg`, название, вес, кол-во ×1, цена справа). Последним пунктом — иконка «+» и ссылка «Добавить ещё» (ведёт в `/catalog`)
3. Разделитель
4. Поле промокода (инпут + кнопка «Применить» справа, уместить в одну строку). При успешном применении — зелёный чип «Промокод −10% активен» с кнопкой «×»
5. Блок «Итог» — построчно, мелким текстом слева и жирным справа:
   - Товары: `2 840 ₽`
   - Скидка: `−284 ₽` (красным/терракотовым)
   - Доставка: `140 ₽` (или `Бесплатно` при сумме от 5000 ₽)
   - Разделитель
   - **Итого:** `2 696 ₽` — крупно `text-2xl font-bold`
6. Финальная кнопка **«Оформить заказ • 2 696 ₽»** — акцентная, во всю ширину, `bg-terracotta text-white py-4 rounded-xl font-semibold`
7. Мелко под кнопкой: «Нажимая кнопку, вы соглашаетесь с [офертой] и [политикой конфиденциальности]»

### Крайние состояния

- **Пустая корзина** — вместо всей формы показать заглушку по центру: иконка пустой сумки, текст «В корзине пока пусто», кнопка «Перейти в каталог» (terracotta). Сайдбар скрыть.
- **Сумма ниже минимальной** (меньше 2000 ₽ на доставке) — плашка-предупреждение красной рамкой в сайдбаре: «До минимального заказа не хватает 460 ₽», и кнопка «Оформить заказ» задизейблена
- **Ошибка отправки** — toast правый-низ красный: «Не удалось оформить заказ. Попробуйте ещё раз»
- **Отправка успешна** — перейти на `/order/[id]` (thank-you) — проектировать отдельно, в этом промпте не нужно

### Взаимодействия и анимации

- Смена Доставка ↔ Самовывоз — плавное fade/slide между блоками через `framer-motion` (пакет `motion`)
- При клике «+ Добавить адрес» — inline-разворачивается форма DaData (accordion), без модалки
- Поля формы — нативные `<input>`, валидация по событию `blur`, ошибка — `text-red-600 text-xs` под полем
- Скролл страницы — sticky-сайдбар остаётся на виду, шаги остаются в своей колонке

## Что НЕ делать

- Не делать переходы между роутами — всё на одной странице `app/checkout/page.tsx`
- Не подключать бизнес-логику (API, валидацию, фактическую отправку) — только UI с заглушками (`const cartItems = [...]`)
- Не переписывать `Header` и `Footer` — использовать существующие
- Не менять дизайн-токены проекта (цвета, шрифты, радиусы)
- Не использовать сторонние UI-киты (shadcn, Radix и т.д.) — только native + `lucide-react` + `motion`
- Эмодзи в иконках мест/оплат не использовать — только `lucide-react`
