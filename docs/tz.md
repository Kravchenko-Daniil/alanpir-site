4) Функциональные требования
4.1 Каталог и карточки

Древовидные категории; у товара: название, фото, состав, аллергенная информация, пищевая ценность, варианты массы (напр. 600/1000 г) с разными ценами — как на референсе. 
piroginomerodin.ru

Метки “Новинка/Хит”, сортировка, быстрый “+ в корзину”.

4.2 Корзина и заказ

Корзина с пересчётом цен, промокодом, указанием суммы скидки/итога.

Оформление заказа: контактные поля (имя, телефон, e-mail), адрес (улица/дом/подъезд/комментарий), дата/окно доставки, способ оплаты (онлайн/при получении), согласие с офертой.

Подтверждение заказа на e-mail клиента и уведомление на почту магазина.

Статусы: создан → подтверждён → готовится → передан курьеру → доставлен → отменён.

Валидация адреса и расчёт стоимости доставки по зонам (логика зон как в разделе “Доставка”: внутри ТТК/до МКАД/за МКАД — цифры укажем из админки). 
piroginomerodin.ru

4.3 Авторизация/регистрация

По e-mail + пароль; восстановление пароля (one-time link).

Опционально: вход по SMS или OAuth (Yandex/Google/Apple) — если потребуется.

4.4 Промокоды

Типы: фиксированная сумма, % скидки, бесплатная доставка, подарок (N-й товар).

Ограничения: срок действия, минимальная сумма, число применений, один раз на пользователя, релевантные категории/товары.

Ввод на странице корзины/чекаута; показ применённой скидки и причины отказа.

4.5 Система лояльности (баллы)

Начисление % от уплаченной суммы (после статуса “доставлен”).

Списание баллов в чекауте (порог/шаг, макс. % корзины).

Отмена/возврат — корректирует балансы.

ЛК: баланс, история операций, правила.

4.6 Отзывы

Форма отправки отзыва (модерация), список с датой/имя/оценка — как на референсе (“Отзывы”). 
piroginomerodin.ru

4.7 Страницы “Доставка и оплата”

Режим работы, каналы оформления (сайт/приложение/колл-центр), способы оплаты (онлайн/курьеру/безнал для юрлиц), калькулятор цены доставки/времени по адресу (минимум — текстовые зоны, улучшение — виджет карты). На референсе — расписаны график и зоны. 
piroginomerodin.ru

4.8 Уведомления

Клиент: письмо “заказ принят”, “заказ оплачен”, “в пути”, “доставлен”, “возврат/отмена”.

Магазин: письмо/telegram о новом заказе и проблемных статусах.

Шаблоны в HTML с логотипом.

Заказы (поиск, статусы, печать накладной/чека, возвраты).

Промокоды (создание, отчёты).

Лояльность (правила, начисления/списания).

Контент: “Доставка/Оплата”, “Акции”, “Отзывы” (модерация), “Блог”.

Параметры доставки (стоимости/минималки/зоны/время) и контакты/телефоны, как на референсе.

5) Нефункциональные требования

Производительность: LCP ≤ 2.5 c на 4G; TTFB ≤ 0.8 c; каталог пагинирован/ленивая загрузка.

Безопасность: HTTPS, CSRF/XSS защита, пароли — bcrypt/argon2; токенизация платежей (карточные данные не храним).

Конфиденциальность: политика cookies/конфиденциальности; соответствие локальным требованиям (GDPR/152-ФЗ — по рынку запуска).

SEO: ЧПУ-URL, мета-теги/OG, микроразметка товара/отзывов, sitemap.xml/robots, скоростные метрики.

Аналитика: events для просмотра карточек, добавления в корзину, чекаута, оплаты, применения промокода.

6) Интеграции

Платежи: hosted checkout (Stripe/ЮKassa/Fondy/PayPal — зависит от юрисдикции). Вебхук → подтверждение оплаты → начислить баллы/открыть чек.

Почта: SMTP/транзакционный сервис (Mailgun/SendGrid/Яндекс 360).

Доставка: базово — собственная логика зон/стоимостей как в “Доставка”; опционально — API карт для расчёта по адресу.

7) Техническая реализация (предложение)

Stack (MVP быстро): WordPress + WooCommerce + плагины (купоны/лояльность), кастомизация темы под референс.

Stack (кастом): Backend (Django/Laravel/Node) + DB (PostgreSQL) + Front (Next.js/Nuxt) + админка (Django Admin/Filament/ custom).

API:

POST /api/checkout/create → создать заказ/платёж, вернуть checkout_url.

POST /api/payment/webhook → приём статусов оплаты (проверка подписи).

POST /api/coupons/apply → валидация промокода.

POST /api/loyalty/redeem / POST /api/loyalty/accrue.

GET /api/me/orders, GET /api/catalog и т.п.

8) Данные/схемы (в общих чертах)

Product: id, title, slug, category_id, images[], description, ingredients, allergens[], nutrition{cal, proteins, fats, carbs}, variants[{weight, price}], flags[new, popular].

Order: id, user_id?, items[{product_id, variant_id, qty, price}], sum, discount, delivery_fee, status, address, comment, payment_id, promo_id?, points_used, points_earned.

Promo: code, type, value, conditions{min_sum, categories, user_limit, date_from/to, max_uses}.

Points: user_id, balance, history[{order_id, delta, reason}].

9) Контент/визуал

Сет баннеров (главная/категории), фотографии товаров (минимум 1200×1200), фавиконки/логотип.

Тексты “О нас/Доставка/Оплата/Акции/Отзывы” (можно сверстать в стиле референса: зоны/график/минималки/время). На референсе указаны расписание и зоны, ссылки на мобильные приложения, контактные телефоны. 
piroginomerodin.ru

10) Тест-кейсы при приёмке

Добавление товаров разных весов; промокод с ограничением по сумме; ошибочный промокод.

Полный цикл оплаты (успех/отказ), проверка вебхука и статуса заказа.

Начисление и списание баллов; возврат заказа и откат баллов.

Письма клиенту/магазину; мобильная вёрстка карточек и корзины.

Расчёт доставки по зонам (минимальная сумма/стоимость/время как в политике доставки).
