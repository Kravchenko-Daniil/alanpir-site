## 1. До первого прода (MVP)

### 1.1 📧 Email-отправка заказов — Web3Forms
**Зачем:** сейчас кнопка «Оформить заказ» никуда не отправляет. Без этого владелец теряет заявки.
**Нужно от клиента:** Access Key с [web3forms.com](https://web3forms.com) (регистрация по email `rizvanovibragim@yandex.ru`).
**Что делаю я:** встроить `fetch('https://api.web3forms.com/submit', ...)` в checkout и форму отзывов, заменить toast на реальную отправку, показать ошибки.
**Переключение на свой backend позже:** меняется одна строка — URL эндпоинта.

### 1.2 📄 Правовые страницы
**Зачем:** по ФЗ-152 сбор ПД (имя, телефон, адрес) без политики конфиденциальности — штраф.
**Что делаю я:** `/privacy/`, `/offer/`, чекбокс «согласие» на всех формах. Возьму стандартный шаблон под ИП.
**Нужно от клиента:** подтвердить ФИО, ИНН, ОГРН, юр. адрес.

### 1.3 ✔️ Валидация форм
**Что делаю я:**
- Маска телефона `+7 (___) ___-__-__`
- Проверка email по regex
- Обязательные поля (имя, телефон)
- Минимальная длина пароля при регистрации
- Сообщения об ошибках под каждым полем
- Disable кнопки пока форма невалидна

### 1.4 🔍 SEO
**Что делаю я:**
- `metadata` на каждую страницу (title + description, уникальные)
- `app/sitemap.ts` → `sitemap.xml`
- `app/robots.ts` → `robots.txt`
- JSON-LD Schema (`Restaurant`, `Menu`, `MenuItem`, `Offer`, `LocalBusiness`)
- Open Graph теги + одна OG-картинка
- `<html lang="ru">` уже есть

### 1.5 📊 Метрика — Яндекс.Метрика + Google Analytics
**Что делаю я:** вставлю код счётчиков с placeholder-ID через `next/script`.
**Нужно от клиента:** зарегистрировать:
- [metrika.yandex.ru](https://metrika.yandex.ru) → получить `YM_ID`
- [analytics.google.com](https://analytics.google.com) → получить `GA_ID`
- Прислать оба ID — подставлю в код.

### 1.6 🎨 Favicon и лого
**Что делаю я:** сверстаю временный favicon (буква «А» в круге, акцентный цвет) + SVG-лого для шапки.
**Нужно от клиента:** если есть фирменный лого АланПир — прислать в SVG/PNG, заменю.

### 1.7 🗺️ Яндекс.Карта на странице самовывоза
**Что делаю я:** заменю плейсхолдер на iframe-виджет Яндекс.Карт с точкой «Москва, Тихвинская 20».

### 1.8 📝 Тексты — «О нас», «Доставка и оплата»
**Нужно от клиента:** чёткие тексты (или хотя бы черновик — 2-3 абзаца на страницу).
**Альтернатива:** напишу свой вариант на основе `docs/content.md`, клиент отредактирует.

### 1.9 🚀 Деплой на alanpir.ru
**Что делаю я:** инструкция по заливке `frontend-next/out/` в ISPmanager. Замена старой версии.
**Нужно от клиента:** логин в ISPmanager REG.RU (или я дам пошаговку, вы сами).

## 3. Вторая итерация (после первого прода)

### 3.1 Фото пирогов
- Положить JPG/WebP в `frontend-next/public/menu/`
- Заполнить поле `image` в `lib/menu.ts` (62 позиции, но актуальны верхние 20-30)
- Оптимизация: сжать до 400×300 (карточка) + 800×800 (модалка)

### 3.2 Настоящий backend (auth + история заказов)
- Выбрать VPS (Timeweb / Selectel / Beget VPS — ~300₽/мес)
- Поднять `backend/` (уже готов: Express + SQLite + nodemailer + DaData)
- Переключить фронт: `NEXT_PUBLIC_API_BASE` вместо Web3Forms
- Работающий личный кабинет, история заказов, сохранённые адреса

### 3.3 DaData — автодополнение адреса
- Получить API-ключ клиента
- Подключить в checkout-модалке адреса
- Валидация, что адрес — в Москве в пределах зоны доставки

### 3.4 Промокоды / скидки
- Нужен документ от клиента: какие коды, сколько процентов, на что действуют, срок, суммируются ли с акциями
- Реализовать логику в бэкенде
- Активировать UI-поле промокода

### 3.5 Эквайринг (онлайн-оплата)
- Выбрать провайдера: ЮKassa / Tinkoff / CloudPayments
- Договор с банком (делает клиент, 1-2 недели)
- Интеграция
- Тестовые платежи
- Возврат денег при отмене
