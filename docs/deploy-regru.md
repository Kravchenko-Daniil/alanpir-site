## Что дальше — выгрузить фронт на REG.RU

Цель: статика на `alanpir.ru`, бэкенд-API остаётся на отдельном VPS.

### Шаги

1. Собрать статику с боевым адресом API:
   ```bash
   cd frontend-next
   NEXT_PUBLIC_API_BASE=<https-URL-бэкенда> npm run build
   # на выходе — папка out/
   ```
2. Залить содержимое `out/` в корень домена через **ISPmanager → Файловый менеджер** (SSH на REG.RU нет).

### Блокеры выгрузки (нужно от Рамиля)

- **Доступ в ISPmanager REG.RU** (B13) — без него залить нельзя.
- **HTTPS у бэкенда.** `alanpir.ru` будет на https, а текущий бэк — `http://46.225.125.127:8080` (временный, на my-hetzner). Браузер заблокирует http-запросы со страницы https (mixed content). Нужен бэкенд по https (постоянный VPS + сертификат) → подставить его URL в `NEXT_PUBLIC_API_BASE` при сборке.

### Напоминание

- `next.config.ts`: `output: 'export'` + `images.unoptimized: true` — не менять (требование REG.RU).
- Бэкенд на REG.RU не работает (нет Node) — только статика фронта.
